# CloudStack TCG — Resource System v2 (Hybrid)

## Overview

The Hybrid resource system creates meaningful deck-building decisions while remaining more forgiving than MTG's pure land system.

## Resource Types

### 1. Generic Credits
- Gain **+1 max generic credit** per turn (caps at 5)
- Generic credits refresh each turn
- Can pay for **colorless costs** on any card
- Represented as: ⚪ or a number

### 2. Domain Credits (from Infrastructure)
- Generated by **tapping Infrastructure cards**
- Required for **domain-specific costs** (colored pips)
- Five domains: 🖥️ Compute, 💾 Data, 🌐 Network, 🧠 Intelligence, 🛡️ Security
- Infrastructure untaps at start of your turn

### 3. Infrastructure Cards
- Play **1 Infrastructure per turn** (like MTG lands)
- Each produces specific domain credits when tapped
- Some produce multiple domains (flexibility) or multiple of one (power)

## Card Cost Notation

Costs are written as: `[Generic][Domain Pips]`

**Examples:**
- `2` = 2 generic credits (any deck can play)
- `2🖥️` = 2 generic + 1 Compute (need Compute infrastructure)
- `🖥️🖥️🖥️` = 3 Compute (heavy Compute commitment)
- `1🖥️🧠` = 1 generic + 1 Compute + 1 Intelligence (multi-domain)
- `4🧠🧠🧠🧠` = 4 generic + 4 Intelligence (late-game Intelligence bomb)

## Infrastructure Examples

### Basic Regions (Single Domain)
```yaml
us-east-1a:
  type: infrastructure
  tap: 🖥️
  flavor: "Compute-optimized availability zone"

us-west-2-data:
  type: infrastructure  
  tap: 💾
  flavor: "S3 and Redshift country"
```

### Dual Regions (Flexibility)
```yaml
us-east-1:
  type: infrastructure
  tap: 🖥️ OR 💾
  flavor: "N. Virginia — where half the internet lives"

eu-west-1:
  type: infrastructure
  tap: 🛡️ OR 💾
  flavor: "Ireland — GDPR headquarters"
```

### Specialty Regions (Power)
```yaml
ml-optimized-region:
  type: infrastructure
  tap: 🧠🧠
  flavor: "GPU instances as far as the eye can see"
  enters_tapped: true  # Drawback for power
```

## Turn Structure (Updated)

### 1. Standup Phase
- Untap all your Infrastructure
- Gain +1 max generic credit (if below cap)
- Refill generic credits to max
- Draw 1 card

### 2. Deploy Phase  
- Play up to 1 Infrastructure from hand
- Play any number of Services/Architectures/Incidents (if you can pay)
- Tap Infrastructure to generate domain credits

### 3. Sprint Phase
- Declare attackers
- Opponent declares blockers
- Resolve combat

### 4. Retrospective Phase
- End-of-turn effects resolve
- Discard down to 7 cards

## Deck Building Rules

- **Minimum 40 cards** (30 non-Infrastructure, 10+ Infrastructure recommended)
- **Maximum 3 copies** of any card (except basic Infrastructure: unlimited)
- No class restrictions — build any domain combination

## Strategic Implications

### Mono-Domain Deck
- 12-14 single-domain Infrastructure
- Consistent domain access
- Can run heavy domain-cost cards (🖥️🖥️🖥️🖥️)
- Example: "Compute Aggro" — fast, scaling services

### Dual-Domain Deck
- 8 Domain A + 6 Domain B Infrastructure
- Mix of dual-tap Infrastructure
- More versatile card pool
- Example: "Data-Intelligence Control" — databases + ML

### Tri-Domain Deck
- Risky but powerful
- Relies on dual-tap Infrastructure
- Can't consistently hit 3+ pip costs
- Example: "Cloud Native" — Compute + Network + Security

### Splash Strategy
- 10+ primary domain, 3-4 splash
- Use generic costs + light splash
- Example: "Compute main, splash 🧠 for Bedrock Claude"

```yaml
  cost: 1  # Just 1 generic!
  stats: 1/1
  keywords: [serverless]

```

### Medium Commitment
```yaml

  cost: 1🖥️  # 1 generic + 1 Compute
  stats: 2/3
  keywords: [scaling]
  effect: "Scaling: Pay 🖥️ to give +1/+1 (max 3x)"
  # Need some Compute infrastructure
```

### Heavy Commitment
```yaml

  cost: 2🖥️🖥️🖥️  # 2 generic + 3 Compute
  stats: 4/5

  # Need heavy Compute investment
```

### Multi-Domain
```yaml
ML Pipeline:
  cost: 2💾🧠🧠  # 2 generic + 1 Data + 2 Intelligence
  stats: 2/4
  effect: "Train: +1 attack each turn. Draw a card when this destroys a service."
  # Need both Data and Intelligence infrastructure
```

### Domain-Hoser
```yaml
Security Audit:
  cost: 1🛡️🛡️
  type: incident
  effect: "Destroy target service that costs 🧠🧠 or more. (AI regulations)"
  # Meta card against Intelligence-heavy decks
```

## Mana Curve Considerations

| Turn | Generic | Infra Played | Typical Plays |
|------|---------|--------------|---------------|
| 1 | 1 | 1 | 1-cost service or just Infrastructure |
| 2 | 2 | 2 | 1🖥️ or 2-cost generic |
| 3 | 3 | 3 | 1🖥️🖥️ or 2🖥️ cards |
| 4 | 4 | 4 | 2🖥️🖥️ or 3🖥️ cards |
| 5 | 5 | 5 | Big plays: 2🖥️🖥️🖥️ or multi-domain |
| 6+ | 5 (cap) | 6+ | Late game bombs, multiple cards/turn |

## Why This Works at 500+ Cards

1. **Deck Identity**: Domain commitment matters — can't just run "best cards"
2. **Build-Around Cards**: "If you control 3+ 🖥️ Infrastructure..." now works
3. **Splash Decisions**: How much do I warp my mana base for that bomb?
4. **Meta Diversity**: Aggro (low curves), Control (high curves), Midrange all viable
5. **Domain Hosers**: Cards that punish specific domains create counterplay
6. **Infrastructure Variety**: 50+ Infrastructure cards with different trade-offs
